title: "Revoke All Active Open API Keys for Your Workspace"
sidebarTitle: "Revoke All Keys"
description: "Immediately and permanently revokes every active Open API key in the workspace. All integrations using any key will begin receiving 401 errors instantly."
---

Use this endpoint to revoke every active API key in your workspace in a single operation. This is intended for emergency scenarios such as a suspected credential compromise, or as the final step of a coordinated key rotation after replacement keys are already in place.

<Note>
  This endpoint uses session cookie authentication (`plangrep_session`). It is used by the Plangrep app's API settings page and does **not** accept a bearer API key.
</Note>

<Warning>
  **This action revokes every active key in your workspace simultaneously and cannot be undone.** All integrations, pipelines, and services authenticating with any API key for this workspace will immediately begin receiving `401 Unauthorized` errors. Only use this endpoint in an emergency (e.g., a suspected key compromise) or as part of a planned, coordinated key rotation where replacement keys are already in place.
</Warning>

## Endpoint

```
POST https://plangrep.com/api/open-api/keys/revoke-all
```

## Authentication

| Method | Value |
|--------|-------|
| Cookie | `plangrep_session=YOUR_SESSION_COOKIE` |

---

## Request Body

No request body required.

---

## Response

**200 OK**

Returns an object containing all key records after revocation.

<ResponseField name="keys" type="OpenApiKey[]" required>
  The full list of key records for the workspace after revocation. All previously active keys will have `revokedAt` set to the timestamp of this operation.

  <Expandable title="OpenApiKey fields">
    <ResponseField name="id" type="string">
      Unique identifier of the key.
    </ResponseField>

    <ResponseField name="label" type="string">
      Human-readable name of the key.
    </ResponseField>

    <ResponseField name="keyPrefix" type="string">
      The prefix used to identify the key.
    </ResponseField>

    <ResponseField name="scopes" type="OpenApiKeyScope[]">
      The permission scopes that were granted to this key. Possible values: `jobs:read`, `jobs:write`, `artifacts:read`.
    </ResponseField>

    <ResponseField name="createdByUserId" type="string">
      ID of the workspace user who originally created this key.
    </ResponseField>

    <ResponseField name="hashVersion" type="string">
      Internal version of the hashing algorithm used to store this key.
    </ResponseField>

    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of when the key was created.
    </ResponseField>

    <ResponseField name="expiresAt" type="string | null">
      ISO 8601 expiration timestamp, or `null` if the key had no expiration.
    </ResponseField>

    <ResponseField name="lastUsedAt" type="string | null">
      ISO 8601 timestamp of the key's most recent successful use before revocation, or `null` if it was never used.
    </ResponseField>

    <ResponseField name="lastUsedIp" type="string | null">
      IP address of the most recent request before revocation, or `null` if the key was never used.
    </ResponseField>

    <ResponseField name="revokedAt" type="string | null">
      ISO 8601 timestamp of when this revocation occurred. Will be non-null for all keys affected by this operation.
    </ResponseField>
  </Expandable>
</ResponseField>

<Tip>
  To rotate keys without downtime, [create replacement keys](/api-reference/workspace-keys/create-key) and update all integrations before calling this endpoint.
</Tip>

<CodeGroup>

```bash cURL
  -H "Cookie: plangrep_session=YOUR_SESSION_COOKIE"
```

Retrieves a single extracted file entity by its ID. The response includes full metadata—classification, processing status, page count—and any artifact URLs generated during extraction. Pass `compact=true` to omit artifact URLs when you only need metadata.

## Endpoint

```
GET https://plangrep.com/api/open/v1/projects/{projectId}/files/{fileId}
```

## Path Parameters

<ParamField path="projectId" type="string" required>
  The unique identifier of the project that owns the file.
</ParamField>

<ParamField path="fileId" type="string" required>
  The unique identifier of the file to retrieve.
</ParamField>

## Query Parameters

<ParamField query="compact" type="boolean">
  When `true`, artifact URL fields are stripped from the response. Useful when you only need metadata.
</ParamField>

<ParamField query="sheets" type="string">
  Comma-separated list of sheet IDs. Restricts the artifact or related data returned to those sheets.
</ParamField>

<ParamField query="fileName" type="string">
  Optional filter — case-insensitive substring match on the file name. Typically used for consistency with list endpoints when sharing query logic.
</ParamField>

<ParamField query="classification" type="string">
  Optional filter by file classification. One of: `plans`, `specifications`, `documents`, `addenda`, `mixed`, `other`.
</ParamField>

<ParamField query="status" type="string">
  Optional filter by processing status string.
</ParamField>

## Response

Returns an object containing the matching file entity.

<ResponseField name="file" type="ProjectFileEntity">
  The requested file entity.

  <Expandable title="ProjectFileEntity fields">
    <ResponseField name="fileId" type="string">
      Unique identifier for this file.
    </ResponseField>
    <ResponseField name="fileName" type="string">
      Original name of the uploaded file.
    </ResponseField>
    <ResponseField name="classification" type="string">
      Plangrep classification of the file: `plans`, `specifications`, `documents`, `addenda`, `mixed`, or `other`.
    </ResponseField>
    <ResponseField name="status" type="string">
      Current processing status of the file.
    </ResponseField>
    <ResponseField name="pageCount" type="integer">
      Number of pages in the file.
    </ResponseField>
    <ResponseField name="artifacts" type="ProjectEntityArtifact[]">
      Array of artifact objects containing URLs to derived outputs, using the canonical artifact type enum. Omitted when `compact=true`.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X GET "https://plangrep.com/api/open/v1/projects/proj_abc123/files/file_001" \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

{
  "file": {
    "fileId": "file_001",
    "fileName": "Architectural_Plans_SetA.pdf",

    "status": "complete",
    "pageCount": 48,
    "artifacts": [
      { "type": "thumbnail", "url": "https://...", "readWith": "plangrep_show_artifact_image" },
      { "type": "source_file", "url": "https://...", "readWith": "plangrep_show_artifact_file" }
    ]
  }

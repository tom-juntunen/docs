---

Retrieves one uploaded specification document by `specId`. The response describes the source document and its artifacts. Use [List Specification Sections](/api-reference/project-reads/list-specification-sections) to inspect the CSI-like sections extracted from this document.

## Endpoint

```
GET https://plangrep.com/api/open/v1/projects/{projectId}/specifications/{specId}
```

## Path Parameters

<ParamField path="projectId" type="string" required>
  The unique identifier of the project that owns the specification document.
</ParamField>

<ParamField path="specId" type="string" required>
  The unique identifier of the uploaded specification document.
</ParamField>

## Query Parameters

<ParamField query="compact" type="boolean">
  When `true`, artifact URL fields are stripped from the response.
</ParamField>

## Response

Returns an object containing the matching specification document.

<ResponseField name="specification" type="ProjectSpecificationEntity">
  The requested specification document row.

  <Expandable title="ProjectSpecificationEntity fields">
    <ResponseField name="specId" type="string">
      Unique identifier for this specification document.
    </ResponseField>
    <ResponseField name="fileName" type="string">
      Original uploaded file name.
    </ResponseField>
    <ResponseField name="classification" type="string">
      File classification for the source document, typically `specifications`.
    </ResponseField>
    <ResponseField name="status" type="string">
      Current processing status for this specification document.
    </ResponseField>
    <ResponseField name="sectionCount" type="integer">
      Number of CSI-like specification sections detected in this document.
    </ResponseField>
    <ResponseField name="artifacts" type="ProjectEntityArtifact[]">
      Artifact objects for this specification document, such as `specification_file`. Omitted when `compact=true`.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X GET "https://plangrep.com/api/open/v1/projects/proj_abc123/specifications/spec_001" \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

## Example Response

```json
{
  "specification": {
    "specId": "spec_001",
    "fileName": "Project_Specifications.pdf",
    "classification": "specifications",
    "status": "complete",
    "sectionCount": 48,
    "artifacts": [
      {
        "type": "specification_file",
        "url": "https://...",
        "readWith": "plangrep_show_artifact_file"
      }
    ]
  }
}
```

## Error Responses

| Status | Code | Description |
|---|---|---|
| `400` | `invalid_request` | Invalid request parameters. |
| `401` | `unauthorized` | Missing or invalid Bearer token. |
| `404` | `open_api_specification_not_found` | No specification document exists for this `specId` in the project. |

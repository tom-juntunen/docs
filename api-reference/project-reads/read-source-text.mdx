title: "Read Paginated Source Text for a Plangrep Project Entity"

Retrieves bounded text content for a source entity identified by a `sourceCatalogUri`. Use this endpoint for sources such as drawing regions, sheets, and document entities. For specification section evidence, use [Get Spec Section](/api-reference/project-reads/get-specification-section) instead so you receive untruncated section text and structured spec fields.

## Endpoint

```
POST https://plangrep.com/api/open/v1/projects/{projectId}/source-text
```

## Path Parameters

<ParamField path="projectId" type="string" required>
  The unique identifier of the project that owns the source entity.
</ParamField>

## Request Body

Content-Type: `application/json`

<ParamField body="sourceCatalogUri" type="string" required>
  Opaque URI identifying the source entity whose text you want to retrieve. Obtain this value from a `sourceCatalogUri` field on a search hit or from the source catalog. Must be at least **1 character**.
</ParamField>

<ParamField body="reviewId" type="string">
  Optional review context identifier. When provided, scopes the text retrieval to sources associated with a specific review.
</ParamField>

<ParamField body="cursor" type="string">
  Pagination cursor. Pass the `nextCursor` value from a previous response to retrieve the next chunk of text. Omit to start from the beginning.
</ParamField>

<ParamField body="maxChars" type="integer">
  Maximum number of characters to return in a single response. Must be between **1** and **50,000**. If the source text exceeds this limit, use `nextCursor` to retrieve the remainder.
</ParamField>

<ParamField body="textKind" type="string">
  Controls the format and scope of returned text. One of:

  | Value | Description |
  |-------|-------------|
  | `text` | Plain text extracted from the source (default). |
  | `markdown` | Text formatted as Markdown, preserving structure where detected. |
  | `page_regions` | Text organized by detected page regions. |
</ParamField>

## Response

Returns a `SourceTextResponse` object.

<ResponseField name="status" type="string">
  Status of the text retrieval operation (e.g. `success`).
</ResponseField>

<ResponseField name="source" type="ProjectSource">
  Structured object describing the source entity.

  <Expandable title="ProjectSource fields">
    <ResponseField name="type" type="string">
      The entity type of this source (e.g. `region`, `doc`, `sheet`).
    </ResponseField>
    <ResponseField name="id" type="string">
      The unique ID of the source entity.
    </ResponseField>
    <ResponseField name="label" type="string">
      Human-readable label for this source.
    </ResponseField>
    <ResponseField name="artifacts" type="ProjectEntityArtifact[]">
      Artifact objects associated with this source, when available.
    </ResponseField>
    <ResponseField name="imageArtifact" type="ProjectEntityArtifact">
      Primary image artifact for this source, when available.
    </ResponseField>
    <ResponseField name="fileArtifacts" type="ProjectEntityArtifact[]">
      File-oriented artifacts for this source, when available.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="sourceCatalogUri" type="string">
  The `sourceCatalogUri` from the request, echoed back.
</ResponseField>

<ResponseField name="text" type="string">
  The retrieved text content, up to `maxChars` characters or the system default. Empty string if no text is available for the requested `textKind`.
</ResponseField>

<ResponseField name="cursor" type="string">
  The cursor value used for this response, echoed from the request.
</ResponseField>

<ResponseField name="nextCursor" type="string">
  Opaque cursor to pass as `cursor` in the next request to continue reading. `null` when there is no more text.
</ResponseField>

<ResponseField name="returnedChars" type="integer">
  Number of characters actually returned in `text` for this response.
</ResponseField>

<ResponseField name="textLength" type="integer">
  Total character length of the full source text. Useful for progress indication.
</ResponseField>

<ResponseField name="hasMore" type="boolean">
  `true` if additional text is available beyond what was returned in this response.
</ResponseField>

<ResponseField name="textKind" type="string">
  The `textKind` that was applied, echoed back from the request or defaulted by the API.
</ResponseField>

## Example Request

```bash
curl -X POST "https://plangrep.com/api/open/v1/projects/proj_abc123/source-text" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "sourceCatalogUri": "plangrep://projects/proj_abc123/regions/region_204",
    "textKind": "text",
    "maxChars": 5000
  }'
```

## Paginating Through Long Source Text

When `hasMore` is `true`, pass `nextCursor` back as `cursor` in a subsequent request to read the next chunk:

```bash
curl -X POST "https://plangrep.com/api/open/v1/projects/proj_abc123/source-text" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "sourceCatalogUri": "plangrep://projects/proj_abc123/docs/doc_018",
    "textKind": "text",
    "maxChars": 5000,
    "cursor": "eyJvZmZzZXQiOjUwMDB9"
  }'
```

Repeat until `hasMore` is `false`.

## Error Responses

| Status | Description |
|--------|-------------|
| `400`  | Invalid request body - `sourceCatalogUri` missing or empty, `maxChars` out of range. |
| `401`  | Missing or invalid Bearer token. |
| `404`  | Project or source entity not found. |

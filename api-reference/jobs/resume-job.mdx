---
title: "Resume a Job Blocked by Insufficient Credits"
sidebarTitle: "Resume Job"
description: "POST /api/open/v1/jobs/{jobId}/resume — resumes a job paused due to insufficient credits after credits have been added."
---

Resume a job that has been paused with a `blocked_insufficient_credits` status. Plangrep automatically pauses processing when your organization runs out of credits mid-job. After adding credits to your account, call this endpoint to restart processing from where it left off — no re-upload is required.

## Authentication

Include your API key as `Authorization: Bearer YOUR_API_KEY` in the request headers.

## Endpoint

```
POST https://plangrep.com/api/open/v1/jobs/{jobId}/resume
```

## Path Parameters

<ParamField path="jobId" type="string" required>
  The unique identifier of the job to resume. The job must currently have a `status` of `blocked_insufficient_credits`.
</ParamField>

## Request Body

This endpoint takes no request body.

## Example Request

```bash
curl --request POST \
  --url https://plangrep.com/api/open/v1/jobs/job_01hy3m8qtn2e5r7k9p0w4x1def/resume \
  --header "Authorization: Bearer YOUR_API_KEY" \
  --header "Accept: application/json"
```

## Response Fields

<ResponseField name="job" type="Job" required>
  The updated job object with the new status reflecting that processing has resumed.

  <Expandable title="Job object fields">
    <ResponseField name="id" type="string" required>
      Unique identifier for the job.
    </ResponseField>

    <ResponseField name="externalId" type="string | null">
      Client-supplied correlation ID, or `null` if none was provided at creation.
    </ResponseField>

    <ResponseField name="status" type="JobStatus" required>
      Updated job status. Will transition to `processing` (or a subsequent non-terminal status) once the resume is accepted.
    </ResponseField>

    <ResponseField name="projectId" type="string" required>
      The ID of the parent project.
    </ResponseField>

    <ResponseField name="limits" type="object" required>
      Processing limits applied to this job.

      <Expandable title="limits fields">
        <ResponseField name="maxDeclaredBytes" type="integer" required>
          Maximum total bytes that may be uploaded to this job.
        </ResponseField>

        <ResponseField name="maxFiles" type="integer" required>
          Maximum number of files that may be uploaded to this job.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="pageCount" type="integer" required>
      Total number of pages submitted for processing.
    </ResponseField>

    <ResponseField name="successfulPageCount" type="integer" required>
      Number of pages that have been successfully processed so far.
    </ResponseField>

    <ResponseField name="billing" type="object" required>
      Updated billing snapshot for the job.
    </ResponseField>

    <ResponseField name="files" type="object[]" required>
      Metadata for each file attached to the job.
    </ResponseField>

    <ResponseField name="resultUrl" type="string | null">
      Pre-signed URL to the normalized result JSON, once the job reaches a completed state.
    </ResponseField>

    <ResponseField name="error" type="object | null">
      Error details if processing subsequently fails, otherwise `null`.
    </ResponseField>

    <ResponseField name="createdAt" type="string" required>
      ISO 8601 timestamp of job creation.
    </ResponseField>

    <ResponseField name="updatedAt" type="string" required>
      ISO 8601 timestamp of the most recent status update.
    </ResponseField>

    <ResponseField name="completedAt" type="string | null">
      ISO 8601 timestamp of when the job reached a terminal state, or `null` if still in progress.
    </ResponseField>

    <ResponseField name="expiresAt" type="string | null">
      ISO 8601 timestamp after which the job and its result are no longer accessible.
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  This endpoint only works on jobs with a current `status` of `blocked_insufficient_credits`. Calling it on jobs in any other status — including active, terminal, or cancelled jobs — will return an error. Verify that you have added sufficient credits to your account **before** calling this endpoint, or the job may be paused again shortly after resuming.
</Note>
